# RostovStat-CRM-System 📊

Информационно-аналитическая система для управления архивными аудиозаписями и статистическими данными регионального отделения. Разработано в качестве **дипломного проекта**.

-----

## 🚀 Основной функционал

  * **Архив аудиозаписей**: Полноценная библиотека с фильтрацией по категориям (*Концентрация, Проф. контент, Совещания*).
  * **Интеллектуальный поиск**: Мгновенный поиск по названиям и описаниям записей.
  * **Личный кабинет**: Персонализированная страница пользователя с расширенной статистикой.
  * **Аналитический дашборд**: Визуализация активности данных с использованием **Chart.js** (динамика загрузок и распределение по категориям).
  * **Оптимизация производительности**: Внедрена система **SWR-кэширования** (*Stale-While-Revalidate*) для мгновенного перехода между страницами без ожидания API.
  * **Безопасность**: Ролевая модель доступа (Admin/Moderator/User), защита от CSRF-атак и контроль сессий.

-----

## 🛠 Технологический стек

### **Frontend**

  * **Vue.js 3** (Composition API)
  * **PrimeVue** (UI компоненты)
  * **Tailwind CSS** (Адаптивная верстка)
  * **Chart.js** (Визуализация данных)
  * **Axios** (HTTP-клиент)

### **Backend**

  * **Python / Django**
  * **Django REST Framework** (RESTful API)
  * **PostgreSQL** (Реляционная СУБД)

## 📦 Установка и запуск

### 1\. Клонирование репозитория

```bash
git clone https://github.com/ваш-логин/RostovStat-CRM-System.git
cd RostovStat-CRM-System
```

### 2\. Настройка Backend (Django)

```bash
cd backend
python -m venv venv
# Для Windows: venv\Scripts\activate
# Для macOS/Linux: source venv/bin/activate 
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

### 3\. Настройка Frontend (Vue.js)

```bash
cd frontend
npm install
npm run dev
```

## 🏗 Архитектура проекта

Проект реализован с соблюдением принципов **Clean Architecture**:

1.  **Разделение ответственности**: Четкое разграничение бизнес-логики, API-слоя и интерфейса.
2.  **Паттерн MVVM**: Использование Model-View-ViewModel на фронтенде для предсказуемого управления состоянием.
3.  **Модульность**: Компоненты системы автономны, что упрощает масштабирование и тестирование.
